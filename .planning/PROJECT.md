# Сервис учета оплаты ЖКУ

## What This Is

Это двустороннее веб-сервисное приложение для отслеживания оплаты жилищно-коммунальных услуг и задолженностей. В системе есть плательщики ЖКУ, которые видят свои квитанции, статусы оплаты и могут имитировать оплату, а также сотрудники компаний ЖКУ, которые видят базовую статистику оплаченных и неоплаченных платежей.

Без входа пользователь может проверить задолженности по лицевому счету: если прошлые периоды оплачены, система показывает подтверждение, если нет - список неоплаченных периодов и видов услуг.

## Core Value

Плательщик должен быстро понять, есть ли у него задолженность по ЖКУ, и увидеть/оплатить свои квитанции в личном кабинете.

### Out of Scope

- Полноценная регистрация и настройка компаний ЖКУ - отложено до следующего этапа, сначала нужен рабочий личный кабинет плательщика и квитанции.
- Реальная интеграция с платежными системами - для дипломного v1 достаточно имитации оплаты с вариантами МИР и СБП.
- Выгрузка расширенной аналитики и отчетов по должникам - будет добавляться после базовой статистики сотрудника.
- Мобильное приложение - веб-версия важнее для первого рабочего результата.
- Публичный показ персональных данных по лицевому счету - без авторизации показываются только статусы прошлых периодов, периоды задолженности и виды услуг.

## Context

Проект создается как дипломная работа на тему "Разработка веб-приложения по отслеживанию оплаты ЖКУ и задолженностей". Нужен понятный сервисный интерфейс, где плательщик не теряется в длинной странице, а быстро переходит между отдельными экранами: главная, вход/регистрация, личный кабинет, квитанции, оплата, профиль.

Первая важная часть продукта - личный кабинет плательщика и квитанции. Служебный кабинет сотрудника компании ЖКУ нужен в минимальном виде: вход и просмотр того, сколько платежей оплачено и сколько осталось неоплаченным. В будущем сотрудник сможет работать с компанией, должниками и более подробной статистикой.

Публичная проверка по лицевому счету должна быть осторожной: она отвечает на вопрос о долгах, но не раскрывает лишние персональные данные. Если задолженности нет, пользователь видит галочку и сообщение, что прошлые периоды оплачены. Если задолженности есть, пользователь видит неоплаченные периоды и виды коммунальных платежей, например вода, газ, электричество.

Визуально приложение должно быть спокойным и практичным: голубая палитра, контрастные понятные блоки, скругленные формы, аккуратные состояния и минимум длинной прокрутки. Лучше больше отдельных страниц, чем одна перегруженная длинная страница.

## Constraints

- **Tech stack**: JavaScript, React, HTML, CSS, Node.js/Express, PostgreSQL и необходимые библиотеки - пользователь хочет этот стек и готов править детали по ходу работы.
- **Database**: PostgreSQL предпочтительнее MySQL для v1, потому что хорошо подходит для связей между плательщиками, квитанциями, услугами и оплатами.
- **Payment**: Реальных платежей нет - только имитация оплаты с визуальными вариантами МИР и СБП.
- **Scope**: Сначала плательщик и квитанции, затем развитие компании ЖКУ и расширенной аналитики.
- **Privacy**: Без авторизации нельзя показывать лишние персональные данные по лицевому счету.
- **UI**: Спокойные голубые цвета, контрастные блоки, скругленные формы, короткие экраны и навигация по страницам вместо длинной прокрутки.
